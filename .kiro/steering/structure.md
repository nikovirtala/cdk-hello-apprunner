# Project Structure

## Directory Layout

```
.
├── src/                    # Source code
│   └── main.ts            # CDK app entry point and stack definition
├── .projen/               # Projen metadata (generated)
├── .github/               # GitHub workflows and configuration
├── .kiro/                 # Kiro AI assistant configuration
│   └── steering/          # AI steering rules
├── cdk.out/               # CDK synthesis output (generated, gitignored)
├── lib/                   # Compiled TypeScript output (generated)
└── node_modules/          # Dependencies (gitignored)
```

## Key Files

- `.projenrc.ts` - Projen configuration (source of truth for project setup)
- `cdk.json` - CDK toolkit configuration
- `package.json` - Generated by Projen, defines scripts and dependencies
- `tsconfig.json` - TypeScript compiler configuration (generated)
- `.eslintrc.json` - ESLint configuration (generated)
- `.prettierrc.json` - Prettier configuration (generated)

## Code Organization

### src/main.ts

Single file containing:
- `AppRunnerStack` class - CDK stack with App Runner service definition
- App instantiation and stack creation
- Stack synthesis

This is a minimal example, so all infrastructure code lives in one file.

## Conventions

- All source code in `src/` directory
- Stack classes extend `Stack` from `aws-cdk-lib`
- Use `CfnOutput` for exposing important values (like service URLs)
- TypeScript compiled to `lib/` directory
- CDK synthesizes CloudFormation to `cdk.out/` directory
